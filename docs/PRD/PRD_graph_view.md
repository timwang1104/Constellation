# PRD：任务依赖图视图（Graph View）

## 1. 背景与目标
在 `prj_kanban` 项目中，任务之间存在前置依赖关系。单纯的看板视图难以直观展示任务间的阻塞链条与关键路径。
本模块旨在提供一个可视化的 DAG（有向无环图）视图，帮助用户（人类与 Agent）理解任务依赖结构，并支持交互式管理依赖关系。

## 2. 核心功能需求

### 2.1 基础展示 (Visualization)
- **节点（Node）**：代表任务（Task）。
  - 展示内容：任务标题、负责人头像/缩写、状态（颜色标记）、ID。
  - 样式区分：
    - 待办（ToDo）：灰色/默认
    - 进行中（Doing）：蓝色/高亮
    - 已完成（Done）：绿色
    - 被阻塞（Blocked）：红色/带警告图标
- **边（Edge）**：代表依赖关系（Dependency）。
  - 方向：箭头指向被依赖的任务（前置 -> 后继），即 A -> B 表示 A 是 B 的前置。
  - 样式：实线表示强依赖。
- **布局（Layout）**：
  - 自动布局：支持从左到右（LR）或从上到下（TB）的层级布局（使用 Dagre 或类似算法）。
  - 避免重叠：节点之间应有适当间距。

### 2.2 交互能力 (Interaction)
- **导航**：
  - 缩放（Zoom In/Out）：支持鼠标滚轮缩放。
  - 平移（Pan）：支持拖拽画布平移。
  - 适配视图（Fit View）：一键适配所有节点到可视区域。
- **操作**：
  - **查看详情**：点击节点，弹出/跳转至任务详情页或侧边栏。
  - **新增依赖**：
    - 方式一：从节点 A 的连接桩（Handle）拖拽连线到节点 B。
    - 方式二：在任务详情中选择前置任务（已在主 PRD 定义，此处侧重图上交互）。
  - **删除依赖**：选中边，按 Delete 键或右键菜单删除。
- **校验**：
  - **环检测**：尝试建立依赖时，若检测到形成环（Cycle），应阻止连线并提示用户。

### 2.3 与架构视图的关联 (Architecture Linking)
- 支持切换或叠加显示任务关联的架构组件（如属于哪个 Service/Module）。
- *（本阶段优先实现任务依赖图，架构视图作为后续扩展）*

## 3. 技术方案建议

### 3.1 前端技术栈
- **框架**：Next.js (React)
- **图形库**：推荐使用 **React Flow** (现更名为 `@xyflow/react`) 或 **AntV X6**。
  - *理由*：React Flow 专为 React 设计，支持自定义节点组件，易于集成 Tailwind CSS，适合流程图/DAG 场景。
- **布局算法**：`dagre` 或 `@elkjs/elk` 用于自动排版。

### 3.2 数据交互
- **API 需求**：
  - `GET /tasks`: 获取任务列表。
  - `GET /dependencies`: 获取依赖关系列表（或包含在任务数据中）。
  - `POST /dependencies`: 创建依赖。
  - `DELETE /dependencies`: 删除依赖。

## 4. 验收标准 (Acceptance Criteria)
1. **渲染正确性**：能正确渲染所有任务节点和依赖连线，无明显错位。
2. **状态可视化**：能一眼识别出哪些任务是被阻塞的（Blocked）。
3. **环检测**：在图上尝试连接 A -> B -> A 时，操作被拦截并提示错误。
4. **交互流畅**：缩放、平移流畅，点击节点能响应。
5. **数据同步**：在图上修改依赖后，刷新看板视图应体现一致的阻塞状态。
