# Graph 页面 UI 设计方案

本文档详细描述 Constellation 项目中 Graph 页面的交互与视觉设计。该页面旨在提供一个直观的视图，用于管理和查看不同 Epic 下的任务依赖关系。

## 1. 布局概览 (Layout Overview)

页面采用**双栏布局 (Two-Column Layout)**，将导航与核心内容分离，确保用户能快速定位上下文并专注于依赖关系的管理。

*   **左侧栏 (Left Column)**：Epic 选择器 (Epic Selector)。
*   **右侧栏 (Right Column)**：任务依赖树 (Task Dependency Tree)。

---

## 2. 左侧栏：Epic 选择器 (Epic Selector)

左侧栏作为主要的导航区域，用于在不同的 Epic（史诗/大颗粒度需求）之间进行切换。

### 2.1 交互模式：滚轮切换 (Roller/Carousel Mode)
*   **视觉隐喻**：采用类似“滚轮”或“老式日期选择器”的 3D 滚轮效果。
*   **操作方式**：
    *   **滚动**：用户可以通过鼠标滚轮、触摸板滑动或点击上下箭头来滚动 Epic 列表。
    *   **选中态**：当前选中的 Epic 位于滚轮的中央（焦点区域），字体放大、高亮显示（如使用主色调或高亮背景），并伴有轻微的吸附效果（Snapping）。
    *   **非选中态**：上下两侧的 Epic 逐渐变小、透明度降低，并在视觉上向后退（Z轴深度变化），营造空间感。
*   **信息展示**：
    *   **Epic 名称**：清晰展示 Epic 的标题。
    *   **进度概览**：在 Epic 卡片上通过微型进度条或环形图展示该 Epic 下任务的完成度。
    *   **状态标识**：使用颜色点或图标标识 Epic 的状态（如：进行中、已完成、阻塞）。

### 2.2 视觉细节
*   **背景**：深色或磨砂玻璃背景，与右侧内容区区分。
*   **动效**：切换时平滑的滚动动画，焦点卡片有缩放和发光效果。

---

## 3. 右侧栏：任务依赖树 (Task Dependency Tree)

右侧栏是页面的核心工作区，展示当前选中 Epic 下的所有任务及其依赖关系。

### 3.1 布局方向：从左向右生长 (Left-to-Right Growth)
*   **生长方向**：
    *   **根节点 (Root)**：通常是 Epic 本身或该 Epic 下的起始任务，位于视图的最左侧。
    *   **子节点/后续任务**：随着依赖关系的延伸，节点向右侧展开。
    *   **层级结构**：
        *   **Level 1**：Epic / 初始任务。
        *   **Level 2**：直接依赖于 Level 1 的任务。
        *   **Level N**：后续依赖任务。
*   **对齐方式**：采用层级布局算法（如 Dagre 或 Elk），确保同一层级的节点在垂直方向上排列整齐，连线尽量减少交叉。

### 3.2 节点设计 (Node Design)
*   **卡片样式**：每个任务节点为一个矩形卡片。
*   **内容包含**：
    *   任务 ID (如 #123)。
    *   任务标题 (截断显示)。
    *   负责人头像 (User/Agent)。
    *   状态指示器 (颜色边框或角标)。
*   **状态可视化**：
    *   **Blocked**：红色高亮，连接线可能变为红色。
    *   **In Progress**：蓝色呼吸效果。
    *   **Done**：绿色或灰色置灰。

### 3.3 交互功能
*   **缩放与平移 (Zoom & Pan)**：
    *   支持无限画布，用户可自由拖拽平移，滚轮缩放查看全局或细节。
*   **小地图 (Mini-map)**：
    *   （可选）右下角提供小地图，方便快速定位。
*   **连线操作**：
    *   贝塞尔曲线连接，线条流畅。
    *   Hover 连线时高亮相关的前后置节点。

---

## 4. 响应式与适配
*   在宽屏模式下，左侧 Epic 选择器常驻。
*   在窄屏或移动端，左侧栏可收起为抽屉（Drawer）或顶部下拉菜单，优先展示右侧依赖图。
